你可能也有过这样的体验：和 AI 聊天时，它能帮你写诗、生成代码，甚至能像老师一样讲解复杂概念——简直聪明得让人惊讶。但当你问它公司最新的销售数据，或者让它帮你预订会议室时，它却像“失忆”了一样，答不上来。为什么会这样？

原因就在于，大多数 AI 模型本身并不直接掌握你的专属信息，它们更多依赖于训练数据里的通用知识。要让 AI 真正能“做事”，它需要两样关键工具：**智能体（Agent）和知识库（Knowledge Base）**。智能体是 AI 的“行动大脑”，它能理解你的指令、调用工具、执行任务；知识库则是 AI 的“记忆仓库”，存储企业数据、文档和规则，让 AI 可以访问和使用最新信息。

不过，在深入了解之前，我们需要先区分几个容易混淆的概念：

- **AI 模型（Model）**：如 GPT-4、Claude，是底层的语言引擎，拥有理解和生成文本的能力，但本身无法主动执行任务。
- **聊天机器人（Chatbot）**：基于 AI 模型构建的对话界面，能回答问题、进行交流，但通常局限于对话场景，无法调用外部工具或执行复杂任务。
- **Agent**：不仅能对话，还能理解指令、制定计划、调用工具（如搜索引擎、数据库、API）并执行任务的智能系统。它是让 AI 从"能聊"到"能做"的关键。

简单来说：模型是"大脑"，聊天机器人是"会说话的助手"，而 Agent 是"既能说又能做的执行者"。


## 智能体：AI 的“大脑”与“手脚”

如果说 AI 模型是拥有海量知识的"大脑"，那么智能体就是让这颗大脑可以真正"动起来"的指挥官。在现代 AI 开发环境如 Claude Code 中，智能体通常以 [Skills](https://chloevolution.com/zh-cn/posts/how-to-use-skills-in-claude-code/) 的形式实现——即标准化、可复用的指令包，定义了 AI 应如何执行特定任务。

它不仅能理解你的指令，还能把一个复杂任务拆解成多个可执行步骤，并决定每一步该如何完成。

### Agent 如何工作？

Agent 的工作流程可以分为四个核心步骤：

1. **感知**：理解用户的指令和意图，识别任务目标。
2. **规划**：将复杂任务拆解成多个可执行的子步骤，并确定执行顺序。
3. **执行**：调用相应的工具和资源，完成每个子步骤。
4. **反馈**：验证执行结果，必要时调整计划或重新执行。

例如，当你要求 Agent"帮我安排下周的团队会议"时，它会：
- 理解你需要安排会议（感知）
- 确定需要检查参与者日历、找到共同空闲时间、发送邀请（规划）
- 依次调用日历 API 查询、选择时间段、发送邮件（执行）
- 确认所有参与者收到邀请并反馈结果（反馈）

Agent最关键的能力，就是**决策和行动**。它之所以能真正“动起来”，靠的就是它可以调用各种工具（Tool Calling）。

工具调用是 Agent 的核心能力，指的是 AI 通过标准化接口调用外部功能的机制。就像人类使用计算器、查阅字典或拨打电话一样，Agent 可以调用各种数字工具来完成特定任务。这让 AI 从“只能说”真正变成“能做”。

这些工具可以按用途分为几大类：

1. **信息获取工具**

   * **网页浏览器**：如 Chrome、Edge，用于实时搜索和获取最新数据。
   * **数据库查询**：如 SQL 数据库、Notion 数据库，帮助智能体读取企业内部信息。
   * **新闻/文档源**：如 RSS、Google News、企业文档库，提供即时参考资料。

2. **办公与协作工具**

   * **日历应用**：Google Calendar、Outlook Calendar，用于安排会议和提醒任务。
   * **邮件客户端**：Gmail、Outlook，用于发送通知或报告。
   * **团队协作平台**：Slack、Teams、Trello，用于发布任务或更新项目状态。

3. **数据处理与分析工具**

   * **电子表格**：Excel、Google Sheets，用于计算、整理和可视化数据。
   * **BI 工具**：Tableau、Power BI，用于生成报告或数据可视化。

4. **执行与自动化工具**

   * **脚本与 API 调用**：Python 脚本、Zapier、Make，用于自动化任务和跨平台操作。
   * **硬件接口**：智能家居设备、打印机、机器人等，让 AI 能影响现实世界。

通过这些能力，智能体让 AI 不再只是"会说话的机器人"，而是真正能与外部世界互动、完成任务的助手。要了解如何在实践中构建和部署这些智能体能力，请参阅[如何使用 Claude Code Skills](https://chloevolution.com/zh-cn/posts/how-to-use-skills-in-claude-code/)。


## 知识库：AI 的“记忆”与“经验”

如果说Agent是 AI 的“大脑与手脚”，那么知识库就是它的“记忆中枢”。知识库存储着海量且精准的信息，是Agent执行任务时的重要参考。它弥补了大模型的两大局限：

1. **知识时效性**
   大模型的训练数据通常有截止日期，所以它可能无法掌握最新的市场数据、新闻或产品信息。而知识库可以实时更新，让 AI 获取最新动态，从而做出更准确的决策。

2. **私有化数据**
   大模型无法直接访问你公司的内部资料或个人笔记。但知识库可以存储这些私密信息，让智能体在需要时安全调用，实现个性化和企业定制化的服务。


从技术角度看，知识库是结构化或非结构化数据的集合，通常包括：
- **文档类**：PDF、Word、Markdown 文件、网页内容
- **数据库记录**：SQL 数据库、NoSQL 数据库中的结构化数据
- **API 响应**：实时从外部服务获取的动态数据
- **多媒体内容**：图片、音频、视频的元数据和描述

这些数据经过向量化处理后，可以被 AI 快速检索和理解，成为 Agent 执行任务时的重要信息来源。

### 知识库与RAG

在技术上，知识库通常结合**RAG（Retrieval-Augmented Generation，检索增强生成）**技术使用。RAG 是一种将**信息检索（Retrieval）**与**生成模型（Generation）**结合的技术。它的核心思想是：在生成回答之前，先从外部知识库或文档中检索相关信息，然后结合这些信息生成更加准确和上下文相关的答案。

RAG 的流程可以分为三个主要步骤：

1. **问题理解与向量化**

   * AI 将用户的提问或指令转换成**向量表示**（embedding）——一串数字序列，用于表示文本的语义含义。这就像给每段文本分配一个"语义指纹"，含义相近的文本会有相似的向量表示。
   * 例如，"如何提升销售业绩"和"怎样增加销售额"虽然用词不同，但它们的向量表示会非常接近。

2. **信息检索（Retrieval）**

   * 系统根据向量相似度，从知识库中找到最相关的文档或数据片段。
   * 这一步通常使用向量搜索（Vector Search）、BM25 等方法来确保检索的高相关性。

3. **生成回答（Generation）**

   * AI 将检索到的内容作为上下文输入大模型，让模型在理解原始指令的基础上生成回答。
   * 生成的内容既包含大模型的语言能力，也整合了最新或专属的知识。

#### 什么是向量数据库？

向量数据库（Vector Database）是专门用于存储和检索高维向量的数据库系统，如 Pinecone、Weaviate、Milvus 等。与传统数据库不同，向量数据库支持**语义相似度搜索**——它不是简单地匹配关键词，而是理解内容的含义。

例如，当你搜索"经济实惠的代步工具"时，向量数据库能找到包含"便宜的汽车"、"二手自行车"等内容的文档，即使这些文档中没有出现"经济实惠"或"代步工具"这些词。

传统的关键词搜索依赖精确匹配，而语义搜索理解查询意图：

| 搜索方式 | 查询："便宜的交通工具" | 能找到的结果 |
|---------|---------------------|------------|
| 关键词搜索 | 只匹配包含“便宜”和“交通工具”的文档 | 范围窄，可能遗漏相关内容 |
| 语义搜索 | 理解“经济型代步方式”的含义 | 能找到“经济型汽车”、“二手自行车”、“公共交通”等相关内容 |

这就是为什么现代知识库系统普遍采用向量化和语义搜索技术——它让 AI 能更准确地找到真正相关的信息。


### Context Window vs Knowledge Base：有什么区别？

很多人会混淆这两个概念，但它们本质上完全不同：

| 特性 | Context Window（上下文窗口） | Knowledge Base（知识库） |
|------|---------------------------|------------------------|
| **定义** | AI 一次对话中能"记住"的内容范围 | 外部存储的信息仓库 |
| **容量** | 有限（如 128K tokens，约 10 万字） | 几乎无限（可存储 GB 甚至 TB 级数据） |
| **访问方式** | 直接加载到模型中，全部内容可见 | 按需检索，只调用相关部分 |
| **成本** | 占用模型计算资源，容量越大成本越高 | 存储成本相对较低 |
| **适用场景** | 当前对话的上下文、最近的交互历史 | 企业文档库、历史记录、专业知识 |

**实际应用中的配合：**
当你向 AI 提问时，Agent 会先从知识库中检索相关信息（如找到 3 篇相关文档），然后将这些文档内容加载到 Context Window 中，结合你的问题生成回答。这样既能访问海量知识，又不会超出模型的处理能力。



## 一体化产品：当“大脑”与“记忆”合二为一

随着 AI 技术的发展，市面上已经出现了将智能体与知识库**深度整合**的一体化产品。这类产品不仅提供信息存储功能，还内置了智能体，使用户无需复杂操作，就能直接与 AI 进行高效交互。

根据知识库的来源，这类产品大致可以分为两类：

1. **私有化知识库**
   以 **NotebookLM** 和 **腾讯 IMA** 为例。当你上传个人文档或企业资料时，这些内容就会成为它的**知识库**。内置的智能体能够理解用户指令，比如“帮我总结”或“回答问题”，并直接从文档中提取信息进行分析和生成答案，实现真正的“即问即答”。

2. **实时动态知识库**
   以 **Perplexity** 为例。它的知识库是**实时、动态**的：当你提出问题时，智能体会即时调用搜索引擎，从互联网上检索最新信息，再结合这些信息生成回答。这意味着无论是最新新闻、实时数据还是最新产品信息，AI 都能立即处理并反馈。

这类一体化产品的出现，让“智能体+知识库”的组合触手可及，大大降低了普通用户和企业使用 AI 的门槛。AI 不再是只能“聊天”的工具，而是可以直接解决实际问题、提供决策支持的智能助手。



## 协同进化：让“大脑”与“记忆”为你工作

智能体和知识库各自强大，但当它们协同工作时，AI 的能力会被放大，让它真正为你“做事”。我们可以通过一个例子来说明：假设你希望 AI 帮你撰写一份公司内部季度报告。整个流程大致如下：

1. **第一步：理解与规划（智能体）**
   当你下达任务后，智能体会先理解你的指令，并进行规划。它会判断：“我需要哪些信息？哪些数据和文档是关键？哪些步骤才能完成这份报告？”这一步相当于 AI 的“头脑风暴”，制定清晰的行动方案。

2. **第二步：检索与获取（知识库）**
   有了行动计划，智能体会去知识库中精准检索相关文件和数据。它可能查阅销售数据表、产品进度文档、内部会议纪要，确保获取的信息既全面又可靠。这一步让 AI 拥有“记忆”，能够访问最新且私密的资料。

3. **第三步：整合与生成（智能体）**
   拿到所需信息后，智能体开始整合和分析，将零散的数据、文档内容和统计信息组织成一份逻辑清晰、条理分明的报告。最终，它生成的内容不仅准确反映公司情况，还符合你的表达要求，真正实现了“能做事”的目标。

通过这种协同机制，智能体与知识库就像“大脑”和“记忆”的完美组合：智能体负责决策和执行，知识库提供信息和经验储备。两者协作，让 AI 不仅能回答问题，还能主动完成复杂任务，成为真正的智能助手。

### 为什么结合知识库能减少 AI 幻觉？

AI"幻觉"（Hallucination）指的是 AI 生成看似合理但实际错误或虚构的内容。知识库能显著减少这种现象，原因在于：

**1. 有据可查**
- 当 AI 基于检索到的真实文档生成答案时，它会引用具体来源
- 例如："根据《2024 年 Q3 销售报告》第 12 页，本季度销售额为 500 万元"
- 这种"有源头"的回答比凭空生成更可靠

**2. 减少猜测**
- 没有知识库时，AI 可能会基于训练数据中的模糊记忆"猜测"答案
- 有了知识库，AI 会明确告知"我在文档中找到了这些信息"或"我没有找到相关信息"

**3. 可验证性**
- 用户可以追溯 AI 引用的文档，验证答案的准确性
- 如果发现错误，可以更新知识库中的源文档，从根本上修正问题

当然，知识库本身的质量至关重要——如果存储的是错误或过时的信息，AI 的回答也会出错。因此，维护高质量的知识库是确保 AI 准确性的关键。

### Agent 可以不依赖知识库工作吗？

答案是：**可以，但能力会受限**。

**不需要知识库的场景：**
- 执行明确的工具调用任务（如查询天气、发送邮件、设置提醒）
- 进行数学计算或逻辑推理
- 调用外部 API 获取实时数据（如股票价格、航班信息）

例如，当你说“帮我查一下明天北京的天气”，Agent 只需调用天气 API，不需要知识库。

**需要知识库的场景：**
- 处理知识密集型任务（如撰写行业报告、回答专业问题）
- 需要历史数据或企业内部信息（如分析过往销售趋势）
- 要求高准确性和可追溯性（如法律咨询、医疗建议）

例如，当你说“根据我们公司过去三年的数据，分析市场趋势”，Agent 必须访问知识库才能完成任务。


## 常见问题（FAQ）

### Agent 和知识库有什么区别？

**Agent（智能体）** 是执行者，负责理解指令、制定计划、调用工具并完成任务。它就像一个能干的助手，知道如何把事情做成。

**知识库（Knowledge Base）** 是信息仓库，存储文档、数据和专业知识。它就像一个图书馆，提供 Agent 执行任务时需要的信息。

简单类比：Agent 是"大脑和手脚"，知识库是"记忆和资料库"。两者结合，AI 才能既"懂"又"会做"。

### 什么时候应该使用 Agent？

当你需要 AI **执行任务**而不仅仅是回答问题时，就需要 Agent：

- **多步骤任务**：如"分析销售数据并生成报告"
- **需要调用工具**：如查询数据库、发送邮件、预订会议室
- **需要决策和规划**：如"根据库存情况制定采购计划"
- **跨系统操作**：如从 CRM 提取数据、在 Excel 中分析、通过 Slack 发送结果

如果任务只是"告诉我某个信息"，简单的 RAG 检索就够了；但如果是"帮我完成某件事"，就需要 Agent。

### 什么时候应该使用知识库？

当你需要 AI 访问**特定领域知识或私有数据**时，就需要知识库：

- **企业内部信息**：员工手册、产品文档、历史项目记录
- **专业领域知识**：医学文献、法律条文、技术规范
- **实时更新的数据**：库存信息、客户数据、市场报告
- **需要准确引用来源**：研究报告、合规文档

如果 AI 的通用知识（训练数据）就能回答问题，不需要知识库；但如果涉及你的专属信息或最新数据，知识库必不可少。

### Agent 和知识库哪个更好？

这不是"哪个更好"的问题，而是**用途不同**：

- **Agent** 解决"怎么做"的问题（执行能力）
- **知识库** 解决"知道什么"的问题（信息储备）

最佳实践是**两者结合**：
- 只有 Agent：能执行任务，但可能缺乏准确信息，容易产生幻觉
- 只有知识库：能提供信息，但无法主动执行复杂任务
- Agent + 知识库：既能准确获取信息，又能高效完成任务

就像一个优秀的员工既要有执行力（Agent），又要有专业知识（知识库）。

### 如何构建 AI 知识库？

构建知识库的基本步骤：

**1. 收集数据源**
- 确定需要包含的文档、数据库、API 等信息源
- 常见格式：PDF、Word、Markdown、网页、数据库记录

**2. 数据预处理**
- 清洗数据：去除无关内容、统一格式
- 分块（Chunking）：将长文档切分成适合检索的片段（通常 500-1000 字）

**3. 向量化（Embedding）**
- 使用 Embedding 模型（如 OpenAI Embeddings、Cohere）将文本转换为向量
- 存储到向量数据库（如 Pinecone、Weaviate、Chroma）

**4. 建立检索系统**
- 实现语义搜索功能
- 配置相似度阈值和返回结果数量

**5. 持续维护**
- 定期更新过时内容
- 监控检索质量，优化分块策略

**工具推荐：**
- 简单场景：LangChain + Chroma（开源免费）
- 企业场景：Pinecone + OpenAI Embeddings（稳定可靠）
- 一体化方案：NotebookLM、腾讯 IMA（无需自建）

### 如何将 Agent 与知识库结合使用？

结合 Agent 和知识库的典型架构：

**1. 任务理解层（Agent）**
- Agent 接收用户指令，理解任务目标
- 判断需要哪些信息和工具

**2. 信息检索层（知识库）**
- Agent 向知识库发起查询
- 知识库返回相关文档或数据片段

**3. 推理与执行层（Agent）**
- Agent 结合检索到的信息进行推理
- 必要时调用其他工具（API、数据库、外部服务）

**4. 结果生成层（Agent）**
- 整合所有信息，生成最终答案或完成任务
- 提供信息来源引用

**实际案例：**
用户："根据我们公司的销售数据，预测下季度需求"
1. Agent 理解任务：需要历史销售数据 + 趋势分析
2. 从知识库检索：过去 12 个月的销售报告
3. Agent 分析数据：识别季节性模式、增长趋势
4. 调用预测工具：使用统计模型生成预测
5. 生成报告：整合分析结果，引用数据来源

### Agent 必须要有知识库吗？

**不是必须的**，取决于应用场景：

**不需要知识库的情况：**
- 任务明确且不依赖特定知识（如"设置明天早上 8 点的闹钟"）
- 可以通过实时 API 获取信息（如"查询今天的天气"）
- 纯逻辑推理或计算任务（如"计算复利收益"）

**需要知识库的情况：**
- 需要访问历史数据或企业内部信息
- 要求高准确性和可追溯性
- 处理知识密集型任务（如专业咨询、研究分析）

**最佳实践：**
现代 AI 系统通常采用**混合架构**：
- 核心知识存储在知识库中（稳定、可控）
- 实时信息通过 API 获取（动态、最新）
- Agent 根据任务需求灵活调用

这样既保证了信息的准确性，又保持了系统的灵活性。

### 什么是 Agentic AI？

**Agentic AI（智能体 AI）** 是指具有**自主性、目标导向和工具使用能力**的 AI 系统。与传统的"问答式 AI"不同，Agentic AI 能够：

**核心特征：**
1. **自主规划**：将复杂目标分解为可执行步骤
2. **工具调用**：主动使用外部工具和 API
3. **持续学习**：从执行结果中调整策略
4. **多轮交互**：在任务执行过程中与用户或系统交互

**与传统 AI 的区别：**

| 特性 | 传统 AI（如 ChatGPT） | Agentic AI |
|------|---------------------|-----------|
| 交互模式 | 一问一答 | 多步骤任务执行 |
| 主动性 | 被动响应 | 主动规划和执行 |
| 工具使用 | 有限或无 | 广泛调用外部工具 |
| 应用场景 | 信息查询、内容生成 | 任务自动化、决策支持 |

**实际应用：**
- **个人助理**：自动安排会议、管理日程、发送提醒
- **企业自动化**：处理客户请求、生成报告、监控系统
- **研究助手**：文献检索、数据分析、假设验证

Agentic AI 代表了 AI 从"工具"向"助手"的进化——它不仅能回答问题，还能主动帮你完成任务。

### Agent 如何从知识库检索知识？

Agent 检索知识的完整流程：

**1. 查询理解**
- Agent 分析用户指令，提取关键信息需求
- 将需求转换为检索查询（可能生成多个查询以覆盖不同角度）

**2. 向量化查询**
- 将查询文本转换为向量表示（embedding）
- 使用与知识库相同的 embedding 模型，确保语义空间一致

**3. 相似度搜索**
- 在向量数据库中计算查询向量与文档向量的相似度
- 常用算法：余弦相似度、欧氏距离
- 返回最相关的 Top-K 个文档片段（如 Top-5）

**4. 重排序（可选）**
- 使用更精细的模型对初步结果重新排序
- 考虑上下文相关性、时效性、权威性等因素

**5. 上下文整合**
- 将检索到的文档片段整合到 Agent 的上下文窗口中
- Agent 基于这些信息进行推理和生成答案

**优化技巧：**
- **混合检索**：结合向量搜索和关键词搜索（BM25）
- **元数据过滤**：先按时间、类别等元数据筛选，再进行语义搜索
- **多跳检索**：根据初步结果进行二次检索，获取更深层信息

### 有了大语言模型，知识库是否已经过时？

**绝对没有**。大语言模型（LLM）和知识库是**互补关系**，而非替代关系：

**LLM 的局限性：**
1. **知识截止日期**：训练数据有时间限制，无法获取最新信息
2. **无法访问私有数据**：不知道你公司的内部文档、客户数据
3. **幻觉问题**：可能生成看似合理但实际错误的内容
4. **无法引用来源**：难以验证信息的准确性

**知识库的不可替代价值：**
1. **实时性**：可以随时更新最新信息
2. **私有性**：安全存储企业专属数据
3. **准确性**：基于真实文档，可追溯来源
4. **可控性**：企业可以管理和审核知识内容

**最佳实践：LLM + 知识库（RAG）**
- LLM 提供语言理解和生成能力
- 知识库提供准确、最新、专属的信息
- 两者结合，既有 LLM 的智能，又有知识库的可靠性

随着 LLM 能力提升，知识库的作用不是减弱，而是**更加重要**——因为 LLM 越强大，就越需要高质量的知识库来确保输出的准确性和可信度。就像人类越聪明，越需要可靠的信息来源一样。
