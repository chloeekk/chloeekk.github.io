如何用Hugo搭建博客並部署到Github page

1. 安裝Hugo

  • Windows：

先安裝choco包管理器，在管理員權限下運行cmd，執行如下命令：

powershell -NoProfile -ExecutionPolicy unrestricted -Command "iex ((new-object net.webclient).DownloadString('https://chocolatey.org/install.ps1'))"

# 設置環境變量
SET PATH=%PATH%;%ALLUSERSPROFILE%\chocolatey\bin

再使用choco 安裝 hugo：

choco install hugo -confirm

  • MacOs：

使用brew 命令安裝：

brew install hugo

  • Linux：

使用snap 命令安裝：

snap install hugo

檢查是否安裝成功：

輸入: hugo version，若出現版本信息則表示安裝成功

2. 新建站點

執行以下命令創建新站點：

hugo new site myblog

這個命令會創建一個名為myblog的文件夾，這就是博客的根目錄。站點目錄結構如下：

├── archetypes
│   └── default.md
├── config.toml         # 博客站點配置文件
├── content             # 博客文章所在目錄
├── data
├── layouts             # 網站佈局
├── static              # 靜態內容
└── themes              # 博客主題
  • archetypes

內容模板文件，其中包含預配置的首選項（日期、標題、草稿等）。可以用自定義的預配置前端字段創建新的原型。

  • config.toml

Hugo 使用 config.toml、config.yaml 或 config.json作為默認網站配置文件。

  • content

存放所有內容文件。

  • data

存儲配置文件。

  • layouts

以 .html 文件的形式存儲模板。

  • static

存儲所有靜態內容，如圖片、CSS、JavaScript 等。

  • themes

你使用的hugo主題。

3. 下載主題

Hugo如果沒有theme是無法啟動的。

新建完站點後可以在官方商店選擇自己喜歡的主題，點擊下載主題會跳轉到主題對應的github。

先把終端路徑調整到myblog文件夾的themes目錄下，再通過git clone命令添加主題（這里以PaperMod主題為例）：

cd themes
git clone https://github.com/adityatelange/hugo-PaperMod.git

將主題添加到配置文件：

$ echo theme = "PaperMod" >> config.toml

4. 啟動博客

在終端輸入：hugo server -D，服務默認會在佔用1313 端口。

執行成功後，會生成一個public 目錄，這個目錄中的內容就是我們所創建的靜態網站的所有內容。

然後在瀏覽器中打開 http://localhost:1313就可以看到你的網站了。

5. 創建文章

通過以下命令新建文章：

hugo new posts/helloworld.md

然後會在content目錄下生成 名為“helloworld.md” 的文件。

所有文章默認放在content文件夾內。如果你有其它的自定義分類目錄，則需要把文章生成到指定目錄下。

每篇文章會包含如下開頭：

---
title: "Hello World"
date: 2022-10-19T21:02:01+08:00
draft: true
---

draft 即 “草稿”，默認值為true，代表此篇內容編譯時將被忽略；改為 false，該篇博客才會被編譯使用。

6. 部署到Github page

①在Github中添加一個空白repository，倉庫名為Github用戶名.github.io，不包含任何內容，如 readme.md文件等。由此得到Github中該repository的網址

②在public 目錄下，依次執行下面的命令：

初始化倉庫

git init

將所有內容添加到git

git add .

提交到git 本地

git commit -m “第一次提交”

關聯到遠程git

注意這裡需要寫你自己的git 地址

git remote add origin https://github.com/Github用戶名/Github用戶名.github.io.git

推送到遠程git

git push origin master`

③至此就將博客內容託管在了Github上了，通過以下地址即可訪問博客：

https://Github用戶名.github.io

後續如果有博客內容更新，則需要再使用hugo 命令生成新的內容，再將新的內容push到Git 倉庫。
