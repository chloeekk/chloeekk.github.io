如何用Hugo搭建博客并部署到Github page

1. 安装Hugo

  • Windows：

先安装choco包管理器，在管理员权限下运行cmd，执行如下命令：

powershell -NoProfile -ExecutionPolicy unrestricted -Command "iex ((new-object net.webclient).DownloadString('https://chocolatey.org/install.ps1'))"

# 设置环境变量
SET PATH=%PATH%;%ALLUSERSPROFILE%\chocolatey\bin

再使用choco 安装 hugo：

choco install hugo -confirm

  • MacOs：

使用brew 命令安装：

brew install hugo

  • Linux：

使用snap 命令安装：

snap install hugo

检查是否安装成功：

输入: hugo version，若出现版本信息则表示安装成功

2. 新建站点

执行以下命令创建新站点：

hugo new site myblog

这个命令会创建一个名为myblog的文件夹，这就是博客的根目录。站点目录结构如下：

├── archetypes
│   └── default.md
├── config.toml         # 博客站点配置文件
├── content             # 博客文章所在目录
├── data
├── layouts             # 网站布局
├── static              # 静态内容
└── themes              # 博客主题
  • archetypes

内容模板文件，其中包含预配置的首选项（日期、标题、草稿等）。可以用自定义的预配置前端字段创建新的原型。

  • config.toml

Hugo 使用 config.toml、config.yaml 或 config.json作为默认网站配置文件。

  • content

存放所有内容文件。

  • data

存储配置文件。

  • layouts

以 .html 文件的形式存储模板。

  • static

存储所有静态内容，如图片、CSS、JavaScript 等。

  • themes

你使用的hugo主题。

3. 下载主题

Hugo如果没有theme是无法启动的。

新建完站点后可以在官方商店选择自己喜欢的主题，点击下载主题会跳转到主题对应的github。

先把终端路径调整到myblog文件夹的themes目录下，再通过git clone命令添加主题（这里以PaperMod主题为例）：

cd themes
git clone https://github.com/adityatelange/hugo-PaperMod.git

将主题添加到配置文件：

$ echo theme = "PaperMod" >> config.toml

4. 启动博客

在终端输入：hugo server -D，服务默认会在占用1313 端口。

执行成功后，会生成一个public 目录，这个目录中的内容就是我们所创建的静态网站的所有内容。

然后在浏览器中打开 http://localhost:1313就可以看到你的网站了。

5. 创建文章

通过以下命令新建文章：

hugo new posts/helloworld.md

然后会在content目录下生成 名为“helloworld.md” 的文件。

所有文章默认放在content文件夹内。如果你有其它的自定义分类目录，则需要把文章生成到指定目录下。

每篇文章会包含如下开头：

---
title: "Hello World"
date: 2022-10-19T21:02:01+08:00
draft: true
---

draft 即 “草稿”，默认值为true，代表此篇内容编译时将被忽略；改为 false，该篇博客才会被编译使用。

6. 部署到Github page

①在Github中添加一个空白repository，仓库名为Github用户名.github.io，不包含任何内容，如 readme.md文件等。由此得到Github中该repository的网址

②在public 目录下，依次执行下面的命令：

初始化仓库

git init

将所有内容添加到git

git add .

提交到git 本地

git commit -m “第一次提交”

关联到远程git

注意这里需要写你自己的git 地址

git remote add origin https://github.com/Github用户名/Github用户名.github.io.git

推送到远程git

git push origin master`

③至此就将博客内容托管在了Github上了，通过以下地址即可访问博客：

https://Github用户名.github.io

后续如果有博客内容更新，则需要再使用hugo 命令生成新的内容，再将新的内容push到Git 仓库。
