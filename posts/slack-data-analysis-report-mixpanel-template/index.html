In the domain of B2B/SaaS, the importance of a comprehensive and real-time data dashboard cannot be overstated. This article serves as a step-by-step guide to constructing a dynamic dashboard for monitoring critical company KPIs. This article offers a template customized to analyze user activities within communication products similar to Slack.

User data

New User Accounts

Determining the influx of new users is a critical component in assessing the growth of a SaaS platform. Follow these steps to obtain valuable insights:

  1. Select “Account Created” Event: Choose the “Account Created” event as the primary metric for the report.
  2. Measure Unique Users: Ensure that the measurement of unique users is enabled for accurate analysis.
  3. Set Time Span to 3 Months: Set the time span to the specific period of interest, in this case, 3 months.
  4. Observation: Upon executing the above steps, the analysis indicates a significant count of 8849 unique users who have created an account in the past 3 months.

Median Number of users by Channel Category

Understanding the distribution of users across different channel categories is crucial for gaining insights into user engagement. Follow these steps to identify the most prominent channel types:

  1. Select the “All Channel(s)” Profile: Begin by selecting the “All Channel(s)” profile as the basis for your report metric.
  2. Measure Median of User Numbers: Calculate the median of the user numbers to obtain a comprehensive understanding of the distribution.
  3. Breakdown by Channel Category: Ensure to categorize the channels into distinct categories such as informative, cross-functional, team-specific, collaborative, mandatory, bots, and fun, among others.
  4. Set Time Span to 30 Days: Analyze the data over a specific 30-day period to capture recent user trends.
  5. Observe User Distribution: Gain valuable insights into the distribution of users across different channel categories, enabling a comprehensive understanding of user preferences and engagement patterns.

Workspaces Data

Understanding the dynamics of workspace creation provides valuable insights into the platform’s growth and user preferences. Follow these steps to analyze the creation of new workspaces within the last 30 days:

  1. Select “Create Workspace” Event: Begin by selecting the “Create Workspace” event as the primary metric for your report.
  2. Measure Total Events: Quantify the total number of events related to workspace creation within the specified time frame.
  3. Breakdown by “Current Plan”: Categorize the new workspaces based on the current plan selected by users, including free, growth, trial, enterprise, and other relevant categories.
  4. Observe New Workspaces Among Different Plans: Gain insights into the distribution of new workspaces across different plans, providing valuable information about user preferences and platform adoption.

Additionally, incorporating the metric of “Average Frequency per Company” enables the assessment of the average number of new workspaces created within a specific time span, segmented by plan type. This comprehensive analysis allows for a deeper understanding of workspace creation patterns and their correlation with different plan types.

Messages Data

Distribution of Messages per Account

Understanding the distribution of messages sent per account is crucial for gauging user engagement and communication patterns. Follow these steps to analyze the distribution of messages sent within the last 30 days:

  1. Select “Send Message” Event: Begin by selecting the “Send Message” event as the fundamental metric for your report.
  2. Measure Frequency Distribution per Company: Analyze the frequency distribution of messages sent per account within the company to obtain a comprehensive understanding of communication patterns.
  3. Set Time Span to 30 Days: Focus the analysis on the specific 30-day period to capture recent messaging trends accurately.
  4. Observe Distribution of Messages Sent per Account: Gain insights into the distribution of messages sent per individual account over the designated time frame, providing valuable information on user engagement and communication habits.

DAU & MAU

Understanding the engagement levels of Daily Active Users (DAU) and Monthly Active Users (MAU) is vital for assessing the platform’s overall performance. Follow these steps to analyze the user activity related to sending messages via the app over a 3-month period:

  1. Select “Send Message” Event: Utilize the “Send Message” event as the primary metric to track user engagement.
  2. Measure “Unique Users - DAU” or “Unique Users - MAU”: Measure the count of unique users for both the Daily Active Users (DAU) and Monthly Active Users (MAU) to understand the frequency and consistency of user engagement.
  3. Set Time Span to 3 Months: Analyze the data over a 3-month period to capture comprehensive user trends and patterns.
  4. Visual Representation: Represent the data using two linear lines on a single chart, reflecting the number of daily and monthly active users who sent messages via the app.
  5. Chart Customization: Optionally, utilize the Stacked Line chart type to visualize the data in a stacked area format, allowing for a comprehensive understanding of the engagement levels of both DAU and MAU over the designated time frame.

Stickiness of Messaging

Assessing the stickiness of messaging within the product is crucial for understanding user engagement and retention. Follow these steps to calculate and analyze the stickiness of messaging over a 3-month period:

  1. Select “Send Message” Event and Measure “Unique Users - DAU”: Utilize the “Send Message” event as the report metric and measure the count of Unique Users - Daily Active Users (DAU) to gauge the daily engagement level.
  2. Select “Send Message” Event and Measure “Unique Users - MAU”: Similarly, use the “Send Message” event as the report metric and measure the count of Unique Users - Monthly Active Users (MAU) to assess the monthly engagement level.
  3. Calculate Stickiness: Determine the stickiness by calculating the ratio of DAU to MAU and multiplying the result by 100. The formula for stickiness is (DAU/MAU)*100.
  4. Set Time Span to 3 Months: Analyze the stickiness trend over a 3-month period to identify any significant changes or patterns.
  5. Visualize Stickiness Trend: Represent the trend of user stickiness to the product over the designated time frame, enabling a comprehensive understanding of user engagement and retention dynamics.

Regular Usage Data

Understanding and analyzing regular usage patterns can provide valuable insights into user behavior and engagement. Follow these steps to define and analyze the regular usage of the application over a 30-day period:

  1. Definition of Regular Usage: For the purpose of this analysis, regular usage is defined as the sequential occurrence of opening the App and sending two messages within a specific time frame.
  2. Sequence of Events Step-by-Step: Create the following sequence of events to define regular usage: a. App Open b. Send Message c. Send Message
  3. Set the Conversion Criteria: Define the Conversion Criteria as “Within 10 weeks,” representing the time window a user has to complete the defined usage sequence from the time of entry.
  4. Set Time Span to 30 Days: Analyze the data over a 30-day period to capture recent trends in regular usage.
  5. Visualize Funnel of Usage Actions: Represent the defined regular usage actions in a funnel chart to gain insights into the conversion rates and user engagement levels.

Furthermore, to gain a deeper understanding of the distribution of usage data, consider converting the funnel chart into a frequency chart, allowing for the customization of the interval size. This approach provides a comprehensive view of the usage patterns and their frequency distribution within the specified time frame.

Onboarding Data

Number of Onboarding Actions

The onboarding process is essential for establishing a smooth user experience within the platform. In this context, onboarding is defined as an admin creating a workspace. Follow these steps to analyze the number of onboarding actions within a 30-day time span:

  1. Sequence of Events Step-by-Step for Onboarding: Create the following sequence of events to define the onboarding process: a. Account Created b. Create Workspace c. Invite Member

  2. Set the Conversion Criteria to Within 30 Days: Define the Conversion Criteria as “Within 30 days,” representing the time window during which a user has to complete the onboarding process from the time of entry.

  3. Set Time Span to 30 Days: Analyze the data over a 30-day period to gain insights into recent onboarding trends and patterns.

  4. Calculate Medium Time to Convert: Determine the medium time to convert, which in this case is 10.2 days, providing valuable insights into the average duration it takes for users to complete the onboarding process.

New User Onboarding Funnel

To understand the effectiveness of onboarding new users, follow these steps to define and analyze the onboarding process within a 30-day time span:

  1. Sequence of Events Step-by-Step for New User Onboarding: a. App Install b. App Open c. Sign Up d. Tutorial Start e. Tutorial Complete

  2. Set the Conversion Criteria to Within 30 Days: Define the Conversion Criteria as “Within 30 days,” representing the time window during which a user has to complete the entire onboarding funnel from the time of entry.

  3. Set Time Span to 30 Days: Analyze the data over a 30-day period to gain insights into recent onboarding trends and patterns.

  4. Visualize the Funnel and Metrics: Obtain the following metrics to assess the efficiency of the onboarding process:

    • Number of users who performed each step
    • Average time taken from the previous step to the current step
    • Conversion rate from the previous step to the current step

This comprehensive analysis enables a detailed understanding of the user onboarding journey, allowing for the identification of potential bottlenecks and areas for improvement.

Annual Spend Data

Annual Spend by Industry

Analyzing the annual spend by industry provides crucial insights into the expenditure patterns of different sectors. Follow these steps to understand the distribution of annual spend within a 30-day time span:

  1. Select “All Company(s)” Profile as Report Metric: Utilize the “All Company(s)” profile as the primary metric for the report to capture a comprehensive view of the entire dataset.

  2. Measure Sum of Annual Spend in USD Dollars: Quantify the sum of the annual spend in USD dollars to comprehend the overall spending trend.

  3. Apply a Filter for “Current Plan” Matching “Is not free”: Incorporate a filter to exclude the data from the “Free” plan as it might not contribute to the annual spend.

  4. Breakdown by “Industry”: Categorize the annual spend data based on different industries such as SaaS, eCommerce, Media, Social, Advertising, and other relevant sectors.

  5. Set Time Span to 30 Days: Analyze the data over a 30-day period to capture recent spending trends accurately.

  6. Visualize Annual Spend by Industry: Gain insights into the distribution of annual spend across various industries, providing valuable information on the expenditure patterns and priorities within different sectors.

Annual Spend Data

Annual Spend by Plan

Analyzing the annual spend by plan type provides essential insights into the spending patterns across different subscription levels. Follow these steps to understand the distribution of annual spend within a 30-day time span:

  1. Select “All Company(s)” Profile as Report Metric: Utilize the “All Company(s)” profile as the primary metric for the report to capture a comprehensive view of the entire dataset.

  2. Measure Sum of Annual Spend in USD Dollars: Quantify the sum of the annual spend in USD dollars to gain an understanding of the total expenditure.

  3. Apply a Filter for “Current Plan” Matching “Is not free”: Incorporate a filter to exclude the data from the “Free” plan, ensuring a focused analysis on the spending patterns of premium plans.

  4. Breakdown by “Current Plan”: Categorize the annual spend data based on different plan types, including Growth, Trial, Enterprise, Platinum, and any other relevant subscription levels.

  5. Set Time Span to 30 Days: Analyze the data over a 30-day period to capture recent spending trends accurately.

  6. Visualize Annual Spend by Plan: Gain insights into the distribution of annual spend across different plan types, providing valuable information on the expenditure patterns and preferences of users across various subscription levels.

User Retention Data

Understanding user retention is crucial for assessing the platform’s ability to maintain an engaged user base. Follow these steps to evaluate user retention within the last 12 weeks:

  1. Access the Retention Report: Initiate the Retention Report feature to gain insights into user retention patterns and trends.

  2. Select “Send Message” Event as Report Metric: Choose the “Send Message” event as the report metric to focus on user engagement and interaction within the platform.

  3. Set the Conversion Criteria to “On or After Each Day” or “On Each Week”: Utilize the “On or After Each Day” criteria for a comprehensive evaluation of the user retention rate, providing a detailed understanding of how long users are retained within the platform. Alternatively, select “On Each Week” to calculate the users that returned in week intervals, offering a periodic assessment of user engagement and retention.

  4. Set Time Span to Last 12 Weeks: Analyze the data over the last 12 weeks to capture recent user retention trends accurately.

  5. Observe User Engagement Over Time: Gain insights into the number of users who performed the “Send Message” action N weeks after their initial engagement, providing valuable information on the platform’s ability to retain an active user base over an extended period.
