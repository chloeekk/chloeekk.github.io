How to Check if Google Analytics is Installed on a Website

Suppose you’re a freelancer or part of an agency, and you’ve recently acquired a new client. You’re interested in determining whether their website has Google Analytics. There are several approaches to do this, and in this blog post, I’ll be sharing various methods to ascertain if Google Analytics is present.

Remember, it’s not necessary to apply all these methods to the same website. If even one method indicates that Google Analytics is active, that’s sufficient confirmation of its installation. The aim of this article is to provide a range of options, allowing you to choose the method that suits you best.

Method 1. Inspect the source code of the page

A widely used method involves opening the website, right-clicking on the website’s background, and choosing ‘View page source’ (the exact wording may vary across different browsers).

Next, look for code snippets associated with Google Analytics. On Windows, press CTRL + F and search for these terms one at a time:

  • gtag. If this appears, the website is utilizing GTAG, a JavaScript library for Universal Analytics (GA3) or Google Analytics 4.

  • analytics.js. This script is specific to Universal Analytics (GA3).

Method 2. Inspect additional files on the website

In certain cases, the Google Analytics (and Google Tag Manager) tracking code might not be visible through ‘View page source’, as some developers opt to place various JavaScript codes in different files of the website.

To investigate this possibility, open your browser’s Developer Tools. I typically use Google Chrome, so the following instructions are tailored for it.

For Windows users, press F12. If you’re using a Mac, press Option + Cmd + J. Then, navigate to the ‘Sources’ section.

Next, press CTRL + SHIFT + F on Windows, or Option + Cmd + F on Mac. This action opens a search feature that scans through all the files loaded on the website, including JavaScript files.

Input the same keywords mentioned earlier in this blog post, one at a time, such as gtag, analytics.js, and gtm.js.

Your objective is to locate the specific file on your website where the tracking code is installed. These files will reveal the tracking code, as illustrated in the screenshot provided below as an example.

Method 3. Inspect the Network Tab within the Developer Tools

For Windows users, press F12. Mac users should press Option + Cmd + J. Then navigate to the ‘Network’ tab. Here, you’ll see a list of all requests made from your website and all the resources downloaded, including those to Google Analytics.

In the search bar, type ‘collect’ and then refresh the page. If Google Analytics is installed on the website, you’ll observe requests that resemble:

  • collect?v=1 (used by Universal Analytics (GA3))
  • collect?v=2 (used by Google Analytics 4)

Method 4. Google Tag Assistant Legacy

Install the Google Tag Assistant Legacy Chrome extension on your browser. Once installed, its icon will appear among your list of extensions. Click on this icon and then select ‘Enable’.

Reload the page where you’re searching for Google Analytics.

If Google Analytics is present, either through GTAG or a Google Tag Manager container on the website, it will be shown there.

Method 5. Google Analytics debugger

Install the GA Debugger extension if you haven’t already. After installation, open the list of your installed extensions and click on the GA Debugger while you are on the website. The page will refresh, and the icon of the extension will change to indicate its active status.

Now, open the developer tools. On Windows, press F12, and on Mac, press Option + Cmd + J. I’ll demonstrate this using Chrome as an example.

Navigate to the ‘Console’, clear all existing entries, and then refresh the page. If you notice entries like the ones shown in the console, it means the GA debugger has detected Google Analytics requests, indicating that Google Analytics is installed on the website.

Method 6. Cookies

When Google Analytics is installed on a website, it creates one or more cookies, including one named _ga.

To check for this, open the developer tools as described in previous sections, and navigate to ‘Application’. Then, select ‘Cookies’ and choose your website’s domain.

In the search field, type in _ga.
