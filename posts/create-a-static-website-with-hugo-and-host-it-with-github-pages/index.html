Are you eager to launch your own blog without getting bogged down in lengthy setup processes? This sentiment resonated with me deeply. My aim was to create a space for sharing my thoughts and musings, yet I was keen on avoiding the tedious and time-consuming setup phase. Upon delving into the current landscape of blogging platforms, I stumbled upon Hugo and quickly realized it might be the perfect solution.

Advantages of Using Hugo and GitHub Pages

  1. Efficient Load Times: Experience the swiftness of Hugo, where your blog’s pages spring to life in the blink of an eye.
  2. Economical Online Presence: Embrace the financial ease of GitHub Pages, a platform that eliminates hosting costs.
  3. Enhanced Team Collaboration: Utilize GitHub’s stellar version control for seamless content updates and team contributions.
  4. Deep Personalization: Dive into Hugo’s rich pool of customization, tailoring your blog’s look and functionality to your exact taste.
  5. SEO-Friendly Design: Benefit from the SEO advantages inherent in Hugo’s static site structure, boosting your online visibility.
  6. Fortified Security: Enjoy peace of mind with the increased security that comes with Hugo’s static site architecture.

Essential Technologies

  • Hugo: A robust engine that transforms simple Markdown into stunning web pages.
    • Rapid Site Construction: Hugo excels in building sites at a remarkable speed.
    • Diverse Theme Selection: Choose from an array of themes to fit your unique style.
  • GitHub Pages: A streamlined hosting solution directly linking your GitHub repo to your live site.
    • Simplified Site Publishing: Effortlessly publish and update your site from your GitHub repository.
    • Personalized Domain Options: Elevate your blog’s professional appeal with a custom domain.
  • Markdown: A user-friendly language for crafting beautifully formatted text.
    • Intuitive Writing Format: Create content easily and efficiently, without the complexities of HTML.
  • Git: A cornerstone tool for source code management and collaborative development.
    • Teamwork Made Easy: Facilitates collaborative writing and editing among contributors.
    • Safe Experimentation: Experiment with new ideas in separate branches, keeping the main project intact.

Prerequisites

Before diving into the world of blogging with Hugo and GitHub Pages, ensure you have the following essentials in place:

  1. Basic Knowledge of Markdown and Git:

    • Familiarity with Markdown for content creation.
    • Understanding of Git for version control and collaboration.

  2. GitHub Account:

    • A GitHub account to host your blog and manage your content. Sign up here.

  3. Text Editor:

    • A reliable text editor like VS Code, Sublime Text, or Atom for writing and editing your content.

  4. Hugo Installed Locally:

    • The Hugo static site generator installed on your computer. Download it here.

  5. Internet Connection:

    • A stable internet connection to access GitHub and publish your blog.

  6. Optional: Custom Domain:

    • If desired, a custom domain for a more personalized blog URL. Providers like GoDaddy or Namecheap can be used.

By setting up these prerequisites, you’ll be well-equipped to start your blogging journey with Hugo and GitHub Pages.

1. Install Hugo on Different Operating Systems

For Windows Users

  • System Version: Windows 10 or later
  • Prompt: Using Windows PowerShell
  • Steps:
    1. Download the Installer: Visit the Hugo Releases page and download the latest version of Hugo for Windows.
    2. Run the Installer: Open the downloaded file and follow the on-screen instructions to install Hugo.
    3. Verify Installation: Open PowerShell and type hugo version to confirm the installation.

For macOS Users

  • System Version: macOS Mojave or later
  • Prompt: Utilizing Homebrew
  • Steps:
    1. Install Homebrew: If not already installed, open Terminal and run /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)".
    2. Install Hugo: In the Terminal, execute brew install hugo.
    3. Confirm Installation: Type hugo version in Terminal to ensure Hugo is properly installed.

For Linux Users

  • System Version: Ubuntu 18.04 LTS or later, Debian-based systems
  • Prompt: Using Terminal
  • Steps:
    1. Add Hugo to the Repository: Open Terminal and add Hugo to your package manager with sudo apt-get install -y hugo.
    2. Update and Install: Update your package lists with sudo apt-get update, then install Hugo with sudo apt-get install hugo.
    3. Verify Installation: Confirm the installation by typing hugo version in the Terminal.

For Arch Linux Users

  • System Version: Arch Linux (latest version)
  • Prompt: Using Terminal
  • Steps:
    1. Use Pacman: Run sudo pacman -Syu hugo in the Terminal to install Hugo.
    2. Confirmation: Type hugo version to verify the successful installation of Hugo.

2. Create a new site

In your command line interface, navigate to the directory where you want your site to be and run: hugo new site my-awesome-site

Replace my-awesome-site with your desired site name. This command creates a new directory with the necessary Hugo structure.

This command will create a folder named my-awesome-site, which is the root directory of the blog. The site directory structure is as follows:

├── archetypes
│   └── default.md
├── config.toml         # Blog site configuration file
├── content             # Directory of blog posts
├── data
├── layouts             # Website layout
├── static              # Static content
└── themes              # Blog Themes
  • archetypes

Content template files with pre-configured preferences (date, title, draft, etc.) New prototypes can be created with custom pre-configured front-end fields.

  • config.toml

Hugo uses config.toml, config.yaml or config.json as the default site configuration file.

  • content

Store all content files.

  • data

Store the configuration file.

  • layouts

Store the template as a .html file.

  • static

Store all static content, such as images, CSS, JavaScript, etc.

  • themes

hugo theme you are using.

3. Choose a Theme

Hugo’s versatility is highlighted in its wide range of themes. Browse the Hugo Themes to find one that resonates with your vision.

Once you’ve selected a theme, add it to your site. For example, if you’re using Git, you can add a theme as a submodule:

cd my-awesome-site
git init
git submodule add https://github.com/user/theme.git themes/theme

Replace the URL with the Git URL of your chosen theme.

4. Add Some Content

Kickstart your site with content. Run: hugo new posts/my-first-post.md

This creates a new Markdown file in your posts directory. Open this file in your text editor to add content.

5. Preview Your Site

To see your site in action, run Hugo’s built-in local server: hugo server -D

Visit http://localhost:1313 in your web browser to preview your new site. The -D flag shows content marked as draft.

6. Customize Your Site

Customize your site by editing the config.toml file in your site’s root directory. Adjust settings like base URL, language, and theme. Feel free to tweak the theme’s templates or add your CSS to personalize your site.

7. Build and Deploy Blog on GitHub Pages

When you’re ready to go live, run: hugo -D

This generates your static site in the public directory.

But if you want to visit your blog publicly, you need to deploy your site to a web server or a platform like GitHub Pages.

Deploying a Hugo site on GitHub Pages can seem daunting at first, but with these detailed steps, you’ll find it surprisingly straightforward. Let’s walk through the process.

Step 1: Prepare Your GitHub Repository

  • Create a New Repository: Head over to GitHub and create a new repository. Name it <yourusername>.github.io, replacing <yourusername> with your actual GitHub username.

  • Clone the Repository: Clone this repository to your local machine using:

git clone https://github.com/<yourusername>/<yourusername>.github.io.git

Step 2: Configure Hugo BaseURL

In your Hugo site’s config.toml file, set the baseURL to https://<yourusername>.github.io. This aligns the site with your GitHub Pages URL.

Step 3: Generate Your Static Site

it’s a familiar action you did before. In the root directory of your Hugo project, run: hugo -D

This command builds your static site in the public directory.

Step 4: Add Your Site to the Repository

  • Move to the Public Directory: Navigate to the public directory generated by Hugo.
  • Initialize a Git Repository: Inside public, initialize a new Git repository:
cd public
git init
  • Add the Remote Repository: Link this repository to your GitHub Pages repository:
git remote add origin https://github.com/<yourusername>/<yourusername>.github.io.git
  • Push to GitHub: Push your commit to GitHub: git push -u origin master

Replace master with main if your default branch is named main.

Step 6: Check Your Live Site

  • Go Live: Your site should now be live at https://<yourusername>.github.io.
  • Verify Your Deployment: Open your browser and navigate to your new site. Check to ensure everything is displaying correctly.

Optional: Automate Deployment with GitHub Actions

For a more streamlined process, consider setting up GitHub Actions to automatically build and deploy your site whenever you push changes to your repository.

Congratulations, you’ve successfully deployed your Hugo site on GitHub Pages! This approach provides a straightforward and cost-effective way to share your content with the world.
