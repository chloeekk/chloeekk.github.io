# 新手必备：最适合Obsidian初学者的5个插件



第一次打开Obsidian插件市场时，我看到1000多个插件，完全懵了。

哪个适合我？哪个最重要？我应该从哪里开始？

结果我装了20个插件，用了一周后发现：只有3个真正在用，其他的都在拖慢速度。

花了2年时间，装了又删，删了又装，我终于知道新手应该从哪些插件开始。

**插件太多，反而是负担**。作为新手，你不需要装很多插件。你需要的是5-8个精选的、简单易用的、能立即提升体验的插件。

无论你是刚[安装好Obsidian](https://chloevolution.com/zh-cn/posts/how-to-install-obsidian/)的新手，还是装了一堆插件但不知道该留哪些，这篇文章都能帮到你。


## 为什么新手需要插件？

### Obsidian核心功能的局限

Obsidian本身只是一个Markdown编辑器。

是的，它很强大。双向链接、图谱视图、本地存储，这些都很棒。但如果你只用核心功能，你会发现：

- **没有日历视图** - 你看不到哪些天写了笔记
- **模板功能太基础** - 不能插入动态日期，不能自动化
- **表格编辑很痛苦** - 手动对齐，添加行列很麻烦
- **没有任务管理** - 只能用简单的checkbox
- **没有绘图工具** - 想画个流程图都要用外部工具

这就是为什么几乎每个Obsidian用户都会装插件。

### 插件如何提升新手体验

插件能让Obsidian从"还行"变成"离不开"。

**立即见效**：
- 装了Calendar，马上有日历视图，一眼看到哪些天有笔记
- 装了Templater，模板变智能，可以自动插入当前日期
- 装了Advanced Tables，表格编辑轻松10倍，自动对齐

**简化操作**：
- 装了Paste URL into Selection，创建链接从3步变成1步
- 装了Kanban，任务管理变得可视化
- 装了Excalidraw，画图不用离开Obsidian

这些插件都有一个共同点：**简单易用，装了就能用，不需要复杂配置**。

### 新手应该装多少个插件？

我的建议：**先装5个，最多10个**。

为什么不是越多越好？

**装太多的问题**：
- 拖慢启动速度
- 插件之间可能冲突
- 增加学习成本
- 大部分都用不上

**正确的策略**：
1. 先装3个必装插件
2. 用1-2周，感受变化
3. 再根据需要装其他插件
4. 一次只装一个，用2周再决定是否保留
5. 定期审查，删除不用的

记住：**少即是多，质量>数量**。

10个真正在用的插件，比50个装了不用的插件强100倍。


## 如何选择适合新手的插件？

不是所有插件都适合新手。

有些插件虽然强大，但学习曲线陡峭。有些插件功能重复，装了反而添乱。有些插件不稳定，可能会出问题。

那么，什么样的插件适合新手？我总结了4个标准。

### 标准1：简单易用

**好的新手插件应该**：
- 安装后就能用，不需要复杂配置
- 界面直观，一看就懂
- 不需要学习新的语法或命令

**举个例子**：

**Calendar** ✅ 适合新手
- 安装后自动显示在右侧边栏
- 点击日期就能创建笔记
- 不需要任何配置

**Dataview** ❌ 不适合新手
- 需要学习类似SQL的查询语言
- 需要理解YAML frontmatter
- 需要写代码才能用

看到区别了吗？Calendar装了就能用，Dataview需要学习。作为新手，优先选择前者。

### 标准2：立即见效

**好的新手插件应该**：
- 装了马上能看到效果
- 解决明显的痛点
- 不需要等待或积累

**举个例子**：

**Advanced Tables** ✅ 立即见效
- 装了之后，表格自动对齐
- Tab键在单元格间跳转
- 马上感觉到编辑表格轻松了

**Dataview** ❌ 需要积累
- 需要先给笔记添加元数据
- 需要积累一定数量的笔记
- 需要时间才能看到价值

新手需要的是"装了就爽"的插件，而不是"用久了才爽"的插件。

### 标准3：稳定可靠

**好的新手插件应该**：
- 下载量高（至少10,000+）
- 活跃维护（最近3个月有更新）
- 评分好（4星以上）
- 不会出问题或冲突

**如何判断**：
1. 在插件市场查看下载量
2. 查看最近更新时间
3. 阅读用户评论
4. 看GitHub的issues数量

**我的建议**：
- 优先选择下载量超过10万的插件
- 避免超过6个月没更新的插件
- 如果评论里很多人说有bug，先不装

### 标准4：真正需要

这是最重要的标准。

**问自己3个问题**：

**1. 我真的需要这个功能吗？**
- 如果答案是"可能有用"，先不装
- 如果答案是"确实需要"，再装

**2. 我会经常用吗？**
- 如果每周用不到一次，不要装
- 只装你会经常用的

**3. 没有它我会不方便吗？**
- 如果没有也无所谓，不要装
- 只装那些"没有会难受"的

**举个例子**：

我刚开始用Obsidian时，看到Excalidraw很酷，就装了。结果3个月都没用过一次，因为我根本不需要画图。

后来我删了它。现在我需要画图了，才又装回来。

**记住**：不要因为"看起来很酷"就装。只装你真正需要的。


## 5-8个最佳新手插件

现在进入重点：我推荐哪些插件？

我把它们分成3个级别：

- **必装（3个）**：新手必须装的，能立即提升体验
- **强烈推荐（3个）**：非常有用，但不是必需的
- **可选（2个）**：适合特定需求的新手

### 必装插件 #1：Calendar

**一句话介绍**：为Obsidian添加日历视图，与Daily Notes完美配合。

**为什么推荐给新手**：

这是我认为新手必装的第一个插件。

- **最简单**：安装后自动显示在右侧边栏，不需要任何配置
- **最直观**：一眼看到哪些天有笔记，有笔记的日期会有圆点标记
- **最实用**：点击日期就能创建/打开Daily Note，不用手动输入日期
- **最流行**：下载量400万+，几乎人人都装

**解决什么问题**：

如果你用[Daily Notes](https://chloevolution.com/zh-cn/posts/obsidian-daily-notes/)记日记或做每日记录，没有Calendar会很不方便：
- 看不到哪些天写了笔记
- 创建Daily Note要手动输入日期
- 想回顾过去的笔记，要在文件夹里翻找

装了Calendar，这些问题都解决了。

**如何安装**：

1. 打开Settings（设置）
2. 点击左侧的"Community plugins"
3. 点击"Browse"按钮
4. 搜索"Calendar"
5. 点击"Install"，然后点击"Enable"

**基本使用方法**：

安装后，Calendar会自动显示在右侧边栏。

- **创建今天的笔记**：点击今天的日期
- **打开过去的笔记**：点击有圆点的日期
- **查看哪些天有笔记**：有笔记的日期会有圆点标记
- **快速跳转**：点击任意日期都能快速跳转


### 必装插件 #2：Templater

**一句话介绍**：智能模板工具，比核心Templates插件强大100倍。

**为什么推荐给新手**：

如果你用过核心的Templates插件，你会发现它太基础了。Templater是Templates的增强版。

- **更强大**：可以插入动态日期、时间、文件名
- **更灵活**：可以运行JavaScript代码（不用担心，简单的功能不需要写代码）
- **学习曲线平缓**：从简单的日期插入开始，逐步学习高级功能
- **社区支持好**：文档详细，示例丰富

**解决什么问题**：

核心Templates只能插入静态内容。但你经常需要：
- 在[模板](https://chloevolution.com/zh-cn/posts/obsidian-templates/)中插入当前日期
- 自动生成文件名
- 根据不同情况显示不同内容

Templater都能做到。

**如何安装**：

1. Settings → Community plugins → Browse
2. 搜索"Templater"
3. Install → Enable
4. 在Templater设置中，指定模板文件夹路径


**基本使用方法**：

创建一个模板文件，比如`meeting-template.md`：

```markdown
# 会议笔记 - <% tp.date.now("YYYY-MM-DD") %>

**时间**：<% tp.date.now("HH:mm") %>
**参与者**：

## 讨论内容

## 行动项

## 下次会议
```

使用模板时，`<% tp.date.now() %>`会自动替换为当前日期和时间。

**使用技巧**：

- **从简单开始**：先学会插入日期，其他的慢慢来
- **查看示例**：官方文档有很多实用的模板示例
- **设置快捷键**：可以设置快捷键快速插入模板

**常用的Templater命令**：
- `<% tp.date.now() %>` - 当前日期
- `<% tp.file.title %>` - 文件名
- `<% tp.date.tomorrow() %>` - 明天的日期



### 必装插件 #3：Advanced Tables

**一句话介绍**：让Markdown表格编辑变得轻松，自动对齐和格式化。

**为什么推荐给新手**：

如果你在Obsidian里编辑过表格，你就知道有多痛苦。

手动对齐、添加行列、调整格式...每次都要小心翼翼，一不小心就乱了。

Advanced Tables解决了Markdown表格编辑的最大痛点。

- **自动对齐**：不用手动调整，表格自动对齐
- **快捷键操作**：Tab键跳到下一个单元格，Enter键创建新行
- **智能格式化**：自动添加分隔符，保持格式整齐
- **提高效率**：表格编辑效率提升10倍

**解决什么问题**：

Markdown表格编辑的痛点：
- 手动对齐很费时间
- 添加行列很麻烦
- 表格一多就乱了
- 编辑时容易出错

**如何安装**：

1. Settings → Community plugins → Browse
2. 搜索"Advanced Tables"
3. Install → Enable

**基本使用方法**：

创建一个表格：

```markdown
| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 25 | 北京 |
```

把光标放在表格内，然后：
- **Tab键**：跳到下一个单元格
- **Shift + Tab**：跳到上一个单元格
- **Enter键**：创建新行
- **自动对齐**：输入内容后自动对齐

**使用技巧**：

- 在表格内使用快捷键，在表格外不会触发
- 可以设置表格样式（左对齐、右对齐、居中）
- 支持表格排序功能


### 强烈推荐 #1：Kanban

**一句话介绍**：看板式任务管理，可视化你的待办事项。

**为什么推荐给新手**：

如果你需要管理任务或项目，Kanban是最直观的方式。

- **可视化**：任务状态一目了然
- **拖拽操作**：直观易用，不需要学习
- **不需要复杂配置**：创建看板，添加卡片，就能用
- **适合多种场景**：任务管理、项目管理、内容创作流程

**解决什么问题**：

纯文本的任务列表有这些问题：
- 任务太多，看不清优先级
- 看不到任务的状态（待办、进行中、完成）
- 缺少可视化，不够直观

**如何安装**：

1. Settings → Community plugins → Browse
2. 搜索"Kanban"
3. Install → Enable

**基本使用方法**：

1. 创建一个新笔记，比如`项目管理.md`
2. 点击左侧边栏的Kanban图标，或使用命令面板
3. 创建列（如：待办、进行中、完成）
4. 添加卡片（任务）
5. 拖拽卡片在列之间移动

**使用技巧**：

- 可以给卡片添加标签和日期
- 可以归档完成的任务
- 可以在卡片中嵌入图片
- 支持Markdown格式

**我的使用经验**：

我用Kanban管理写作流程：Ideas → Drafting → Editing → Published。每篇文章是一张卡片，拖到不同列表表示不同状态。


### 强烈推荐 #2：Excalidraw

**一句话介绍**：手绘风格的图表工具，直接在Obsidian中画图。

**为什么推荐给新手**：

有时候，文字不如图表直观。

- **不需要学习复杂的绘图软件**：界面简单，工具栏一目了然
- **手绘风格**：视觉友好，不会太正式
- **可以链接到笔记**：画的图可以链接到其他笔记
- **适合快速草图**：流程图、思维导图、架构图都能画

**解决什么问题**：

- 想画图但不想用外部工具
- 需要流程图、思维导图来整理想法
- 想要可视化复杂的概念
- 需要快速草图记录灵感

**如何安装**：

1. Settings → Community plugins → Browse
2. 搜索"Excalidraw"
3. Install → Enable

**基本使用方法**：

1. 创建Excalidraw笔记（命令面板搜索"Excalidraw"）
2. 使用工具栏画图：
   - 矩形、圆形、箭头
   - 文字、线条
   - 手绘笔
3. 完成后可以导出为PNG或嵌入笔记

**使用技巧**：

- 使用快捷键提高效率（R=矩形，C=圆形，A=箭头）
- 可以链接到其他笔记（创建可点击的元素）
- 支持多种导出格式
- 可以设置画布背景色

### 强烈推荐 #3：Paste URL into Selection

**一句话介绍**：选中文字，粘贴URL，自动创建Markdown链接。

**为什么推荐给新手**：

这个插件超级简单，但超级实用。

- **没有学习曲线**：装了就会用
- **节省大量时间**：创建链接从3步变成1步
- **符合直觉**：就像在Word里添加链接一样
- **提高效率**：特别适合整理网页收藏和引用资料

**解决什么问题**：

在Markdown中创建链接很麻烦：
1. 要记住`[text](url)`的语法
2. 要先输入文字，再添加链接
3. 容易出错

**如何安装**：

1. Settings → Community plugins → Browse
2. 搜索"Paste URL into Selection"
3. Install → Enable

**基本使用方法**：

超级简单，只需3步：

1. **选中文字**：比如"Obsidian官网"
2. **复制URL**：复制https://obsidian.md
3. **粘贴**：按Ctrl/Cmd + V

自动变成：`[Obsidian官网](https://obsidian.md)`

**使用技巧**：

- 适合整理网页收藏
- 适合引用资料来源
- 大大提高添加链接的效率
- 也可以用于内部链接


### 可选插件 #1：Linter

**一句话介绍**：自动格式化笔记，保持一致性。

**为什么推荐**：

如果你的笔记格式不一致，或者从其他工具导入了笔记，Linter能帮你自动整理。

- 自动修复格式问题
- 统一标题格式
- 清理多余空行
- 修正拼写错误

**何时需要**：

- 你的笔记格式不一致
- 你从Notion、Evernote等工具导入笔记
- 你想要自动化格式整理
- 你有一定的Markdown基础

**如何安装**：

Settings → Community plugins → Browse → "Linter"

**使用提示**：

- 需要配置规则（可以用默认规则）
- 建议先了解Markdown格式规范
- 可以设置保存时自动运行
- 适合有一定基础的新手


### 可选插件 #2：Editor Syntax Highlight

**一句话介绍**：代码块语法高亮，提高代码可读性。

**为什么推荐**：

如果你在笔记中记录代码，这个插件能让代码更易读。

- 提高代码可读性
- 支持多种编程语言
- 轻量级，不影响性能
- 安装后自动生效

**何时需要**：

- 你在笔记中记录代码
- 你是技术用户或程序员
- 你需要更好的代码阅读体验
- 你学习编程时做笔记

**如何安装**：

Settings → Community plugins → Browse → "Editor Syntax Highlight"

**使用提示**：

- 安装后自动生效，不需要配置
- 支持大多数编程语言
- 可以自定义配色主题
- 只在代码块中生效，不影响普通文本


## 新手应该避免的插件

推荐完适合新手的插件，我还要告诉你：哪些插件不适合新手。

不是说这些插件不好，而是它们对新手来说太复杂、太高级，或者容易造成问题。

### 太复杂的插件：Dataview

**为什么不推荐给新手**：

Dataview是Obsidian最强大的插件之一，但它真的不适合新手。

- **需要学习类似SQL的查询语言**：你要写代码才能用
- **需要理解YAML frontmatter**：要给笔记添加元数据
- **学习曲线陡峭**：新手可能要花几周才能上手
- **需要积累**：需要有一定数量的笔记才能看到价值

**举个例子**：

用Dataview查询所有标签为"工作"的笔记：

```dataview
TABLE file.ctime as "创建时间"
FROM #工作
SORT file.ctime DESC
```

看到了吗？你需要学习这种查询语法。对新手来说，这太难了。

**什么时候再考虑**：

- 你已经用Obsidian 3个月以上
- 你有大量笔记（至少100+）
- 你需要复杂的查询和统计
- 你愿意花时间学习

**新手替代方案**：

- 使用核心的Search功能
- 使用标签过滤笔记
- 使用文件夹组织笔记
- 这些对新手来说够用了

### 功能重复的插件

**不要同时装多个任务管理插件**

我见过有人同时装了：
- Kanban
- Tasks
- Checklist
- Day Planner

结果呢？每个都用一点，每个都没用好，最后还搞混了。

**为什么不要这样做**：

- **功能重复**：它们都是做任务管理的
- **造成混乱**：不知道该用哪个
- **增加学习成本**：每个都要学习
- **可能冲突**：插件之间可能不兼容

**正确的做法**：

1. 选一个任务管理插件（我推荐Kanban）
2. 用2周，熟悉它
3. 如果不满意，再换另一个
4. 不要同时装多个

**同样的道理适用于**：

- 不要同时装多个日历插件
- 不要同时装多个模板插件
- 不要同时装多个绘图插件

记住：**一个功能，一个插件**。

### 不稳定的插件

**如何判断插件是否稳定**：

在装插件之前，检查这些指标：

**1. 下载量**
- 低于1,000：可能不稳定
- 1,000-10,000：需要谨慎
- 10,000+：通常比较可靠
- 100,000+：非常可靠

**2. 最近更新时间**
- 超过6个月没更新：可能被放弃了
- 3-6个月：需要注意
- 最近3个月有更新：活跃维护

**3. 评分和评论**
- 评分低于3星：有问题
- 评论里很多人说有bug：不要装
- 评论里说和其他插件冲突：谨慎

**4. GitHub issues**
- 未解决的issues很多：可能有问题
- issues长期无人回应：可能被放弃

**风险**：

装不稳定的插件可能导致：
- Obsidian崩溃
- 笔记丢失或损坏
- 与其他插件冲突
- 浪费时间排查问题

### 过度定制的插件

**什么是过度定制的插件**：

有些插件有几十个设置选项，需要大量配置才能用。

**为什么不适合新手**：

- **配置复杂**：光是设置就要花很长时间
- **选择困难**：不知道该怎么配置
- **容易出错**：配置错了可能不工作
- **学习成本高**：要理解每个选项的作用

**举个例子**：

有些插件的设置页面有：
- 20+个开关选项
- 10+个文本输入框
- 5+个下拉菜单
- 还有高级设置、自定义CSS等

对新手来说，这太overwhelming了。

**新手应该选择**：

- 安装后就能用的插件
- 默认设置就很好用的插件
- 设置选项少的插件
- 有"推荐配置"的插件

少装几个，但每个都真正在用，这才是正确的策略。


## 插件安装和管理指南

知道了该装哪些插件，现在我来教你如何安装和管理它们。

如果你是第一次装插件，跟着这个指南一步步来，不会出错。

### 如何启用社区插件

在安装任何社区插件之前，你需要先启用社区插件功能。

**首次启用步骤**：

1. 打开Obsidian
2. 点击左下角的设置图标（齿轮图标）
3. 在左侧菜单找到"Community plugins"
4. 你会看到一个安全警告
5. 点击"Turn on community plugins"按钮
6. 阅读风险提示，点击确认

**安全提示**：

Obsidian会显示这样的警告：

> 社区插件由第三方开发者创建，不是Obsidian官方开发的。这些插件可以访问你的文件系统和网络。请只安装你信任的插件。

这个警告很重要。记住：
- 只安装下载量高的插件
- 只安装你真正需要的插件
- 定期检查和清理插件

想了解更多关于社区插件的信息，可以查看[Obsidian官方插件文档](https://help.obsidian.md/Extending+Obsidian/Community+plugins)。

启用后，你就可以开始安装插件了。

### 如何安装插件

**安装步骤**（以Calendar为例）：

1. **打开插件市场**
   - Settings → Community plugins
   - 点击"Browse"按钮

2. **搜索插件**
   - 在搜索框输入"Calendar"
   - 可以看到搜索结果列表

3. **查看插件详情**
   - 点击插件名称
   - 查看插件描述、作者、下载量
   - 查看最近更新时间
   - 阅读用户评论

4. **安装插件**
   - 点击"Install"按钮
   - 等待安装完成（通常几秒钟）

5. **启用插件**
   - 安装完成后，点击"Enable"按钮
   - 插件就开始工作了

### 如何更新插件

插件会不断更新，添加新功能或修复bug。定期更新插件很重要。

**更新步骤**：

1. Settings → Community plugins
2. 点击"Check for updates"按钮
3. 如果有更新，会显示"Update"按钮
4. 点击"Update"更新单个插件
5. 或点击"Update all"更新所有插件

**更新建议**：

- **每周检查一次**：养成定期检查的习惯
- **阅读更新日志**：了解新功能和变化
- **重要插件先测试**：更新后测试一下是否正常工作
- **不要跳过更新**：更新通常包含bug修复和安全补丁

**如果更新后出问题**：

1. 先禁用插件，看问题是否消失
2. 如果是插件问题，可以在GitHub上报告
3. 或者暂时回退到旧版本（需要手动操作）

### 如何禁用和删除插件

如果你发现某个插件不需要了，或者想暂时禁用它，很简单。

**禁用插件**：

1. Settings → Community plugins
2. 在已安装插件列表中找到要禁用的插件
3. 点击插件旁边的开关，关闭它
4. 插件立即停止工作，但不会被删除

**删除插件**：

1. 先禁用插件（关闭开关）
2. 点击插件旁边的垃圾桶图标
3. 确认删除
4. 插件被完全删除

**我的建议**：

- **先禁用，再删除**：禁用几天，确认不需要再删除
- **不要一次删除太多**：一次删一个，观察是否有影响
- **定期清理**：每月审查一次，删除不用的插件

## 常见问题解答

### Q1: 我应该先装哪个插件？

从这3个必装插件开始：

1. **Calendar** - 最简单，最实用，装了立即见效
2. **Templater** - 提升模板功能，使用频率高
3. **Advanced Tables** - 解决表格编辑痛点

装完这3个，用1-2周，感受变化。如果觉得不错，再考虑装Kanban、Excalidraw等其他插件。


### Q2: 插件是免费的吗？

大多数插件都是免费的。

**免费的**：
- 几乎所有社区插件都是免费的
- 由开发者自愿开发和维护
- 不需要付费就能使用全部功能

**付费的**（官方服务）：
- **Obsidian Sync**：$4/月，官方同步服务
- **Obsidian Publish**：$8/月，发布笔记到网站

这两个是官方付费服务，不是插件。如果你不需要同步或发布，完全可以免费使用Obsidian和所有社区插件。

### Q3: 插件会拖慢Obsidian吗？

可能会，取决于你装了多少个。

**影响因素**：
- **插件数量**：装得越多，越慢
- **插件类型**：实时处理的插件（如Dataview）更耗资源
- **笔记库大小**：笔记越多，某些插件越慢
- **设备性能**：老电脑更容易感觉到变慢

**如何避免**：
- 保持插件数量在10个以内
- 禁用不常用的插件
- 避免装功能重复的插件
- 定期检查启动时间

**我的经验**：
我装了15个插件，Obsidian启动时间大约3秒。如果你装了30个，可能要10秒以上。

### Q4: 如何知道插件是否安全？

检查这4个指标：

**1. 下载量**
- 10万+：非常可靠
- 1万-10万：通常可靠
- 1千-1万：需要谨慎
- 低于1千：不建议新手装

**2. 最近更新时间**
- 最近3个月有更新：活跃维护
- 3-6个月：需要注意
- 超过6个月：可能被放弃

**3. 评分和评论**
- 4星以上：质量好
- 3-4星：一般
- 低于3星：有问题

**4. 是否开源**
- 大多数社区插件都是开源的
- 可以在GitHub上查看源代码
- 开源意味着代码可以被审查

**我的建议**：
优先选择下载量10万+、最近3个月有更新、评分4星以上的插件。

### Q5: 可以在移动端使用这些插件吗？

部分支持。

**我推荐的插件中**：

✅ **支持移动端**：
- Calendar
- Templater
- Advanced Tables
- Kanban
- Paste URL into Selection

⚠️ **部分支持**：
- Excalidraw（功能有限）
- Linter（可用但不太方便）

❌ **不支持**：
- Editor Syntax Highlight（移动端不需要）

**如何确认**：
在安装插件前，查看插件描述，通常会说明是否支持移动端。

**我的建议**：
如果你主要在移动端使用Obsidian，装插件前先确认移动端兼容性。

### Q6: 装了插件后如何卸载？

很简单，两步搞定。

**卸载步骤**：

1. **先禁用**
   - Settings → Community plugins
   - 找到要删除的插件
   - 点击开关，关闭它

2. **再删除**
   - 点击插件旁边的垃圾桶图标
   - 确认删除
   - 插件被完全删除

**我的建议**：
- 先禁用几天，确认不需要再删除
- 不要一次删除太多，一次删一个
- 删除后重启Obsidian，确保干净

### Q7: 插件之间会冲突吗？

可能会，但不常见。

**什么情况会冲突**：
- 功能重复的插件（如多个任务管理插件）
- 修改相同功能的插件
- 某些插件的已知兼容性问题

**如何避免冲突**：
- 不要装功能重复的插件
- 一次装一个，测试后再装下一个
- 阅读插件文档，看是否有已知冲突

**如果出现冲突**：
1. 禁用最近安装的插件
2. 逐个启用，找出冲突的插件
3. 选择其中一个，删除另一个
4. 或者在GitHub上报告问题

**我的经验**：
用了2年，只遇到过2次冲突。只要不乱装，基本不会有问题。

### Q8: 我应该装多少个插件？

我的建议：**新手5-8个，最多10个**。

**为什么不是越多越好**：
- 装太多会拖慢速度
- 增加学习成本
- 容易冲突
- 大部分都用不上

**正确的策略**：
- 先装3个必装插件
- 用1-2周，确认需要
- 再根据需求装其他插件
- 定期审查，删除不用的

**我的插件数量**：
我现在装了15个插件，每个都有明确的用途，每周至少用一次。

**记住**：10个真正在用的插件，比50个装了不用的插件强100倍。
