# Obsidian模板指南：从入门到精通，附7个实用模板


在使用Obsidian记笔记时，你是否发现自己在重复做同样的事情？每次创建读书笔记都要手动输入"书名、作者、阅读日期"这些字段；每次写Daily Note都要重新设置相同的标题结构；每个项目笔记都需要从头开始搭建"目标、任务、进度"的框架。

这些重复工作不仅浪费时间，还容易导致笔记格式不统一。当你的vault里有几百条笔记时，格式混乱会让信息检索变得困难。

Obsidian的模板功能可以彻底解决这个问题。通过使用模板，你可以：
- 一键创建具有统一结构的笔记
- 自动插入当前日期、时间和笔记标题
- 保持所有同类型笔记的格式一致
- 将重复工作时间减少80%以上

本文将详细介绍：
- 如何设置和使用Obsidian的模板功能
- 核心Templates插件与Templater插件的区别
- 7个可以直接使用的实用模板
- 模板设计的最佳实践


## 什么是Obsidian模板？

模板是预先设计好的笔记结构，可以帮助你快速创建具有相同格式的笔记。当你需要创建新笔记时，只需插入模板，Obsidian就会自动填充预设的内容和结构。

在Obsidian中，模板特别适合以下场景：

- **读书笔记**：统一的书籍信息、摘要和思考结构
- **项目管理**：一致的项目概述、目标和任务列表
- **Daily Notes**：每日笔记的固定结构和反思问题
- **周报/月报**：定期总结的标准格式
- **Zettelkasten卡片**：符合卡片盒笔记法的标准格式
- **文献笔记**：学术研究中的文献记录模板

使用模板的优点有：
1. **节省时间**：不再需要重复输入相同的结构和格式，专注于内容本身
2. **保持一致性**：所有同类型的笔记都使用统一的格式，便于后续检索和整理
3. **提高效率**：配合Dataview等插件，统一的格式让数据查询和汇总变得简单


## 如何使用Obsidian核心Templates插件

Obsidian内置了Templates插件，这是官方提供的模板解决方案。它简单易用，能满足大多数用户的基本需求。

### 第一步：启用Templates插件

1. 打开Obsidian设置（点击左下角齿轮图标，或使用快捷键 `Cmd/Ctrl + ,`）
2. 在左侧菜单中选择"核心插件"
3. 找到"模板"（Templates），点击开关启用
4. 启用后，左侧菜单会出现"模板"设置项

![Obsidian core plugins settings page showing the Templates plugin toggle in enabled state](obsidian-templates-plugin-enable.png)

### 第二步：配置模板文件夹

启用插件后，需要指定一个文件夹来存放所有模板文件：

1. 在"模板"设置页面，找到"模板文件夹位置"（Template folder location）选项
2. 输入文件夹路径，例如 `Templates`

![Obsidian Templates plugin settings page displaying the template folder location configuration field](obsidian-templates-folder-settings.png)

**重要提示**：Obsidian不会自动创建这个文件夹。当你在设置中输入文件夹路径后，Obsidian只是在底层配置里记住了"以后如果用户要调用模板，去这个名字的文件夹里找"。如果这个文件夹在你的vault里实际上并不存在，模板功能将无法使用。

3. 手动创建模板文件夹：
   - 在文件浏览器中，右键点击vault根目录
   - 选择"新建文件夹"（New folder）
   - 输入你在设置中指定的文件夹名称（如 `Templates`）

**建议**：将模板文件夹放在vault的根目录，便于管理和访问。


### 第三步：创建你的第一个模板

让我们创建一个简单的读书笔记模板：

1. 在模板文件夹中创建新文件，命名为 `读书笔记模板.md`
2. 输入以下内容：

```markdown
---
书名:
作者:
出版年份:
阅读日期: {{date}}
标签: #读书笔记
---

- **书名**：
- **作者**：
- **出版年份**：
- **阅读日期**：{{date}}

## 💡 核心观点

1.
2.
3.

## 📝 重要摘录

>

## 🤔 个人思考

## 🔗 相关笔记

```

3. 保存文件

这个模板使用了 `{{date}}` 变量，插入模板时会自动替换为当前日期。

![Obsidian file explorer showing the Templates folder containing multiple template files including book notes template](obsidian-templates-folder-structure.png)


### 第四步：使用模板

有两种方式可以插入模板：

**方法一：通过命令面板**
1. 创建或打开一个笔记
2. 按 `Cmd/Ctrl + P` 打开命令面板
3. 输入"插入模板"（Insert template）
4. 选择你想使用的模板

![Obsidian command palette showing the "Insert template" command with a list of available templates](obsidian-insert-template-command.png)


**方法二：设置快捷键**
1. 打开设置 → 快捷键
2. 搜索"插入模板"
3. 设置你喜欢的快捷键，例如 `Cmd/Ctrl + T`

### Templates插件支持的变量

Templates插件提供了几个实用的变量，可以在模板中自动填充信息：

| 变量 | 说明 | 示例输出 |
|------|------|----------|
| `{{title}}` | 当前笔记的标题 | "我的笔记" |
| `{{date}}` | 当前日期 | 2026-05-23 |
| `{{time}}` | 当前时间 | 14:30 |
| `{{date:YYYY-MM-DD}}` | 自定义日期格式 | 2026-05-23 |
| `{{date:YYYY年MM月DD日}}` | 中文日期格式 | 2026年05月23日 |

**日期格式说明**：
- `YYYY` = 四位年份
- `MM` = 两位月份
- `DD` = 两位日期
- `HH` = 24小时制小时
- `mm` = 分钟
- `ss` = 秒

例如，`{{date:YYYY-MM-DD HH:mm}}` 会输出 `2026-05-23 14:30`

## Templater插件：更强大的模板解决方案

如果你觉得核心Templates插件的功能不够用，可以尝试社区插件Templater。它提供了更强大的功能，包括JavaScript脚本、用户自定义函数、动态内容生成等。

### Templater vs Templates：该选哪个？

| 特性 | Templates（核心插件） | Templater（社区插件） |
|------|----------------------|----------------------|
| 安装方式 | 内置，直接启用 | 需要从社区插件安装 |
| 学习曲线 | 简单，5分钟上手 | 中等，需要学习语法 |
| 变量支持 | 基础变量（日期、标题等） | 丰富的内置变量和函数 |
| 自定义能力 | 有限 | 支持JavaScript脚本 |
| 动态内容 | 不支持 | 支持（如随机内容、API调用） |
| 适用场景 | 日常基础需求 | 高级自动化需求 |

**建议**：
- 如果你是新手，或只需要基础的模板功能，使用核心Templates插件即可
- 如果你需要复杂的自动化、动态内容生成，或有编程基础，可以尝试Templater

### 如何安装Templater插件

1. 打开Obsidian设置 → 社区插件
2. 关闭"安全模式"（如果还未关闭）
3. 点击"浏览"按钮
4. 搜索"Templater"
5. 点击"安装"，然后点击"启用"

![Obsidian Templater plugin settings page showing template folder location and other configuration options](obsidian-templater-settings.png)

### Templater示例

以下是一个使用Templater的Daily Notes模板示例，展示了它比核心插件更强大的功能：

```markdown
---
日期: <% tp.date.now("YYYY-MM-DD") %>
星期: <% tp.date.now("dddd") %>
周数: 第<% tp.date.now("WW") %>周
天气: ☀️
---

# 📅 <% tp.date.now("YYYY年MM月DD日 dddd") %>

## 🌤️ 每日一言

<% tp.web.daily_quote() %>

## ✅ 今日任务

<% tp.file.cursor(1) %>

## 📝 笔记

<% tp.file.cursor(2) %>

---

[[<% tp.date.now("YYYY-MM-DD", -1) %>|← 昨天]] | [[<% tp.date.now("YYYY-MM-DD", 1) %>|明天 →]]
```

**Templater的独特功能**：
- `<% tp.file.cursor() %>` - 设置光标位置，插入模板后光标会自动定位
- `<% tp.date.now("YYYY-MM-DD", -1) %>` - 日期计算，-1表示昨天，1表示明天
- `<% tp.web.daily_quote() %>` - 调用网络API获取每日名言
- 支持条件判断、循环等编程逻辑

### 常用Templater函数

| 函数 | 说明 | 示例 |
|------|------|------|
| `tp.date.now()` | 当前日期时间 | `<% tp.date.now("YYYY-MM-DD") %>` |
| `tp.date.tomorrow()` | 明天的日期 | `<% tp.date.tomorrow("YYYY-MM-DD") %>` |
| `tp.date.yesterday()` | 昨天的日期 | `<% tp.date.yesterday("YYYY-MM-DD") %>` |
| `tp.file.title` | 当前文件标题 | `<% tp.file.title %>` |
| `tp.file.cursor()` | 设置光标位置 | `<% tp.file.cursor(1) %>` |

如果你想深入学习Templater，可以查看官方文档：[https://silentvoid13.github.io/Templater/](https://silentvoid13.github.io/Templater/)


## 模板设计的最佳实践

在使用模板的过程中，我总结了一些让模板更好用的方法，分享给你。

### 1. 保持简洁，避免过度设计

刚开始用模板时，我也犯过"过度设计"的错误，恨不得把所有信息都塞进去。

**不好的做法**：
```markdown
## 📚📖📝 读书笔记 📝📖📚
═══════════════════════════════
【书名】：
【作者】：
【出版社】：
【ISBN】：
【页数】：
【定价】：
【装帧】：
【出版年份】：
═══════════════════════════════
```

**好的做法**：
```markdown
## 📖 读书笔记

**书名**：
**作者**：
**阅读日期**：{{date}}
```

后来发现，只保留真正需要的字段，模板反而更容易坚持使用。过多的装饰和字段会让人产生"填写负担"。

### 2. 使用一致的命名和结构

在所有模板中保持一致的风格：
- 统一使用emoji或不使用
- 统一日期格式（如都用 `YYYY-MM-DD`）
- 统一标题层级（如一级标题用于笔记标题，二级标题用于章节）
- 统一元数据字段名称

这样做的好处是，当你的笔记越来越多时，格式统一会让检索和整理变得轻松很多。

### 3. 为未来的自己设计

设计模板时，多想想"三个月后的自己"会怎么用这些笔记：
- **检索性**：使用统一的标签和元数据，便于后续搜索
- **可扩展性**：预留足够的空间添加新内容
- **兼容性**：如果使用Dataview等插件，确保字段格式兼容

### 4. 定期回顾和优化

我的习惯是每个月回顾一次模板使用情况：
- 哪些字段从来不填？考虑删除
- 哪些内容经常需要手动添加？考虑加入模板
- 哪些模板使用频率很低？考虑合并或删除

模板不是一次性设计好就不变的，而是需要根据实际使用情况不断调整。

### 5. 建立模板文件夹结构

当模板数量增多时，建议按类型组织：

```
Templates/
├── 工作/
│   ├── 会议记录.md
│   ├── 项目管理.md
│   └── 周报.md
├── 学习/
│   ├── 读书笔记.md
│   ├── 课程笔记.md
│   └── 文献笔记.md
└── 个人/
    ├── Daily Notes.md
    ├── 周回顾.md
    └── 目标追踪.md
```


## 常见问题解答

### 1. 模板插入后，变量没有被替换怎么办？

**原因**：可能是变量语法错误，或使用了Templater语法但没有安装Templater插件。

**解决方案**：
- 核心Templates插件使用 `{{date}}` 语法
- Templater插件使用 `<% tp.date.now() %>` 语法
- 确认你使用的语法与安装的插件匹配

### 2. 可以在模板中使用Dataview查询吗？

可以。在模板中添加Dataview代码块，插入后会正常执行：

````markdown
```dataview
TABLE 状态, 优先级
FROM #项目管理
WHERE 状态 = "进行中"
```
````

### 3. 如何为不同的文件夹设置默认模板？

使用Templater插件的"Folder Templates"功能：
1. 安装并启用Templater
2. 在Templater设置中找到"Folder Templates"
3. 添加文件夹路径和对应的模板

例如：`Projects/` 文件夹自动使用"项目管理模板"。

### 4. 模板可以包含双向链接吗？

可以。在模板中使用 `[[]]` 语法，插入后会创建链接：

```markdown
## 相关笔记
- [[]]
- [[]]
```

### 5. 如何在模板中设置光标位置？

**Templates插件**：不支持光标定位

**Templater插件**：使用 `<% tp.file.cursor() %>`

```markdown
# {{title}}

<% tp.file.cursor(1) %>

## 笔记内容

<% tp.file.cursor(2) %>
```

插入模板后，光标会定位到第一个位置，按Tab键跳转到下一个位置。

### 6. 可以在移动端使用模板吗？

可以。Obsidian移动端完全支持Templates插件和Templater插件。操作方式：
1. 打开命令面板（点击左上角命令图标）
2. 搜索"插入模板"
3. 选择模板

### 7. 模板文件本身会被搜索到吗？

会。如果不想在全局搜索中看到模板文件，可以：
1. 在设置 → 文件与链接 → 排除的文件中添加模板文件夹
2. 或在模板文件的YAML中添加 `publish: false`

### 8. 如何分享模板给其他人？

最简单的方式：
1. 将模板文件复制到对方的vault
2. 或将模板内容发送给对方，让其手动创建

如果你有多个模板要分享，可以：
- 创建GitHub仓库存放模板
- 使用Obsidian的同步功能

模板功能看起来简单，但用好了能大幅提升笔记效率。

从启用Templates插件、创建第一个模板，到使用本文提供的7个实用模板，再到了解Templater这个更强大的选择——你现在已经掌握了在Obsidian中使用模板的完整方法。

我的建议是：先从一个简单的模板开始，比如Daily Notes或读书笔记。用一段时间后，根据实际需要调整字段和结构。模板不是一次性设计好就不变的，而是在使用中不断优化的。

当你发现某个笔记类型总是重复相同的结构时，那就是创建新模板的信号。慢慢地，你会建立起一套适合自己的模板库，让记笔记这件事变得更轻松、更高效。
